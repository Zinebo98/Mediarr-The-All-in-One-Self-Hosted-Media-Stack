# 🎬 Mediarr — The All-in-One Self-Hosted Media Stack

A self-hosted, GPU-accelerated media platform running on **Proxmox VE**, automated end-to-end from acquisition to playback. Built around Jellyfin with a custom toolchain for reliable library syncing, hardware transcoding, and metadata/catalog automation.

> This repository documents the architecture, the engineering problems I solved, and the custom tooling I wrote. Configuration is sanitized — no secrets or API keys are included.

---

## Overview

The goal: a "click play, it streams" media experience for local content **and** an on-demand streaming tier, fully automated, running on a single Proxmox host alongside other workloads. The system handles acquisition, organization, metadata enrichment, hardware-accelerated transcoding, and curated discovery — with custom scripts filling the gaps the off-the-shelf tools left.

**Highlights**
- 🖥️ Intel iGPU passed through to a VM for **hardware transcoding + HDR→SDR tone mapping**
- ⚡ Custom **targeted library-scan** script that avoids slow, hang-prone full scans
- 🤖 Fully automated acquisition pipeline (full *arr suite + dual debrid backends)
- 🗣️ **AI-generated subtitles** via Whisper (speech-to-text)
- 🎫 Self-service requests (Seerr) + automated quality profiles (Recyclarr)
- 🗂️ Curated discovery via Trakt / TMDB catalogs merged into mixed Jellyfin collections
- 🧱 ZFS storage layout with a deliberate redundancy/cost trade-off

## Architecture

![Mediarr architecture](docs/architecture.png)

A request flows through five stages — **Request → Find & Grab → Download → Store → Serve** — running on a Debian VM (Docker) on a Proxmox host, with the Intel iGPU passed through to Jellyfin for hardware transcoding. A separate streaming tier (AIOStreams / AIOMetadata / Gelato) layers on-demand "Instant" libraries.

### Storage (ZFS)
| Pool | Layout | Purpose |
| --- | --- | --- |
| Flash | single NVMe | VM root + appdata (fast) |
| Vault | 2-disk stripe | media library (re-acquirable; no redundancy by design) |
| Main  | single disk | data that actually needs backup, kept separate |

The redundancy choice is deliberate: media is re-downloadable, so it lives on a capacity-optimized stripe; the one irreplaceable dataset is isolated on its own pool.

## The Stack

| Layer | Tools |
| --- | --- |
| **Media server** | Jellyfin — hardware-accelerated transcoding (Intel QSV) |
| **Acquisition (\*arr)** | Sonarr · Radarr · Prowlarr (indexers) · Bazarr (subtitles) · Flaresolverr |
| **Requests** | Seerr (self-service request portal) |
| **Quality automation** | Recyclarr (TRaSH-Guides quality-profile sync) |
| **Download backends** | TorBoxarr · Decypharr — TorBox / debrid via qBittorrent + SABnzbd shims |
| **Streaming tier** | AIOStreams · AIOMetadata (+Redis) · Gelato (catalogs → "Instant" libraries) |
| **AI** | Whisper — automatic subtitle generation from audio (speech-to-text) |
| **Discovery** | Trakt + TMDB catalogs, merged into mixed movie/series collections |
| **Custom tooling** | Targeted library-scan + library-cover generator (this repo) |

All services run as Docker containers across a few compose stacks on the media VM. The pipeline is fully hands-off: a request in Seerr → grabbed by Sonarr/Radarr (quality profiles maintained by Recyclarr) → fetched through a debrid backend → subtitles auto-generated by Whisper and fetched by Bazarr → discovered in Jellyfin within minutes by the targeted-scan script.

## Engineering Highlights

### 1. Intel iGPU passthrough for hardware transcoding
The media VM had no GPU, so 4K HDR transcodes ran in software — a single file's trickplay generation once took **10.5 hours**. I:
- Enabled the iGPU in BIOS alongside the discrete GPU (iGPU multi-monitor),
- Bound it to `vfio-pci` on the Proxmox host,
- Passed it through to the VM (q35), swapped the cloud kernel for one with the `i915` driver, and installed the Intel media driver,
- Configured Jellyfin QSV with **VPP tone mapping** (the OpenCL path fails on the LinuxServer image).

Result: decode + tone-map + encode all run on the iGPU; CPU usage during a 4K HDR transcode dropped from ~600% to background levels.

→ Full writeup: [`docs/optional/hardware-transcoding.md`](docs/optional/hardware-transcoding.md)

### 2. Targeted library scanning (`jellyfin-reconcile.py`)
Full library scans were slow and hung at ~88% on the streaming "Instant" libraries. Instead of relying on them, I wrote a reconcile script that:
- Queries Jellyfin's API for everything already indexed,
- Diffs it against what's actually on disk,
- Triggers an **instant, per-file targeted scan** (via the Targeted Scans plugin) only for new items — never a full scan.

Runs on a 5-minute cron as a self-healing safety net. See [`scripts/jellyfin-reconcile.py`](scripts/jellyfin-reconcile.py).

→ Details: [`docs/optional/targeted-scanning.md`](docs/optional/targeted-scanning.md)

### 3. Metadata & catalog automation
Trakt and TMDB catalogs are enriched via AIOMetadata and surfaced in Jellyfin through Gelato. Merged catalogs combine movie + series sources into single mixed collections (e.g. "Top Rated" with both), and a custom library-cover generator produces a consistent set of branded library tiles.

→ Details: [`docs/10-streaming-tier.md`](docs/10-streaming-tier.md)
